Features:
This project uses a multitude of features based off the fundamental pillars of a good DJ transition. These are rhythmic synchronization, harmonic compatibility, and spectral density. Most of the features extracted can be computed using librosa internal library.

Focusing on rhythmic density a good transition typically has a single coherent pulse or beat. A bad transition exhibits blurred beats or double onset peaks as it changes from the old song to the new song. To evaluate this, the onset strength envelopes and tempograms. These features track the timing of percussive transients, such as kick drums and snares, allowing the model to identify misaligned beats. 

To analyze harmonic capability, one can incorporate Chroma features, which can map audio energy into twelve bins representing semitones. Using theses features the model could potentially detect harmonic dissonance, specifically when two tracks in incompatible keys overlap. 
Finally, Mel-Frequency Cepstral Coefficients (MFCCs) were used to monitor the spectral energy and texture. MFCCs allow the model to perceive muddy mixes, where overlapping low end frequencies create a distorted sound. 

In addition, a couple standard features that are taught in audio machine learning labs are incorporated into the code.
